# 交通标志识别系统功能说明书

## 1. 系统概述

交通标志识别系统是面向软件综合课程设计的 Web 应用。系统围绕交通标志检测与识别、标志信息解析、位置查询、统计分析、管理员维护和违规提醒构建完整功能链路。

系统支持图片、视频和摄像头三类输入方式。识别结果会被解析为业务字段并保存到数据库，用户可在前端查询历史记录、查看统计报表，管理员可维护交通标志字典和处理违规记录。

## 2. 用户角色

### 2.1 管理员

管理员用于系统维护和全局数据管理，主要功能包括：

- 查看所有检测记录
- 管理交通标志字典
- 查看违规预警台账
- 查看统计分析和日报
- 查看审计日志
- 删除无效记录并清理关联文件

### 2.2 驾驶员/普通用户

普通用户用于完成识别和查询，主要功能包括：

- 使用检测工作台进行图片、视频或摄像头识别
- 查看本人历史检测记录
- 查询交通标志信息
- 接收违规提醒

## 3. 功能模块说明

### 3.1 登录与权限控制

系统提供登录、注册和角色识别能力。前端根据用户角色展示不同菜单入口，后端接口继续作为权限边界。

主要能力：

- 用户注册与登录
- Token 鉴权
- 管理员/普通用户菜单区分
- 路由守卫检查登录状态

### 3.2 交通标志检测工作台

检测工作台是系统核心入口，支持图片、视频和摄像头三种模式。

图片检测：

- 上传交通标志图片
- 填写当前车速、驾驶动作等参数
- 设置置信度阈值
- 展示检测结果图和结构化识别详情
- 根据限速或禁令规则触发违规提醒

视频检测：

- 上传视频文件
- 按帧间隔抽取关键帧
- 对抽样帧进行交通标志识别
- 汇总检测结果和违规事件

摄像头检测：

- 调用浏览器摄像头预览
- 支持单帧抓拍检测
- 支持连续监控模式
- 发现违规后即时提醒

### 3.3 标志信息解析

系统将模型输出的类别、置信度和检测框转换为业务可读信息。

解析字段包括：

- 标志编码
- 标志名称
- 标志类型
- 标志含义
- 建议速度
- 路段与方向
- 位置描述
- 经纬度
- 检测置信度
- 检测框坐标

### 3.4 历史检测记录

系统会保存检测结果，便于后续查询和追溯。

主要能力：

- 按时间、标志类型、违规状态筛选
- 查看检测来源：图片、视频或摄像头
- 查看原图、结果图和识别详情
- 查看违规原因和风险等级
- 管理员可删除无效记录

### 3.5 违规预警台账

当系统判断驾驶行为违反识别到的交通标志规则时，会生成违规记录。

支持的典型违规：

- 超速行驶
- 违停
- 禁止掉头场景
- 禁止鸣笛场景

违规记录字段：

- 违规类型
- 风险等级
- 违规说明
- 当前状态
- 处理人
- 处理时间

管理员可将违规记录标记为已处理，也可重新打开记录。

### 3.6 标志位置查询

系统支持通过关键词、路段、方向、类型和位置半径查询交通标志。

主要能力：

- 按关键词检索标志
- 按路段和方向筛选
- 按经纬度和半径查询附近标志
- 返回标志含义、位置描述和建议速度

### 3.7 标志字典管理

标志字典用于连接模型类别和业务含义，是识别结果解析的基础。

管理员可维护：

- 标志编码
- 标志名称
- 标志类型
- 标志含义
- 限速值
- 路段、方向、位置描述
- 经纬度
- 模型 class id 映射

支持 CSV 批量导入导出，便于初始化和维护数据。

### 3.8 数据分析与报告

系统可对检测记录和违规记录进行统计分析。

统计内容包括：

- 检测总量
- 违规事件数量
- 标志类型分布
- 近 30 天检测趋势
- 违规类型占比
- 违规趋势

系统支持生成日报并导出 CSV。

### 3.9 审计日志

系统记录关键操作，便于追踪管理员维护行为。

审计内容包括：

- 登录行为
- 字典新增、编辑、删除
- 记录删除
- 违规处理
- 数据导入导出

## 4. 后端接口概览

后端基于 FastAPI，主要接口类别如下：

- 用户接口：注册、登录、获取当前用户
- 检测接口：图片检测、视频检测、摄像头帧检测
- 记录接口：检测记录查询、详情查看、删除
- 违规接口：违规记录查询、处理、重新打开
- 字典接口：标志字典增删改查、导入导出
- 查询接口：按条件查询标志、查询附近标志
- 统计接口：仪表盘统计、趋势分析、日报生成
- 系统接口：模型状态、审计日志、文件清理

## 5. 前端页面概览

前端基于 Vue3 和 Element Plus，主要页面包括：

- 登录页
- 注册页
- 检测工作台
- 历史记录页
- 标志查询页
- 标志字典管理页
- 数据分析页
- 审计日志页

页面设计以课程设计演示为目标，强调入口清楚、流程连续、结果可解释。

## 6. 数据存储说明

系统使用 SQLite 保存结构化数据，文件系统保存上传图片、视频和检测结果图。

核心数据表包括：

- 用户表
- 标志字典表
- 检测记录表
- 违规记录表
- 审计日志表
- 系统配置表

检测记录会保存原图路径、结果图路径、检测详情 JSON、来源类型、位置描述和用户名等信息。

## 7. 模型与演示模式

系统支持两种识别方式：

- 真实模型模式：当检测到 `best.pt` 模型文件时，可加载 YOLOv8 模型进行真实识别。
- 演示模式：当模型不可用时，系统使用 mock 检测结果，保证课程设计答辩时功能链路可以稳定演示。

这种设计可以兼顾可扩展性和演示稳定性。

## 8. 测试与运行

后端 smoke 测试：

```powershell
python -B scripts\smoke_backend.py
```

前端构建：

```powershell
cd traffic-sign-ui
npm run build
```

安全检查：

```powershell
cd traffic-sign-ui
npm audit --audit-level=moderate
```

## 9. 答辩演示建议

建议按以下顺序演示：

1. 登录系统，说明管理员和普通用户入口。
2. 打开检测工作台，上传图片并填写当前车速。
3. 展示识别结果图、结构化详情和违规弹窗。
4. 切换视频或摄像头检测入口，说明多输入方式。
5. 查看历史检测记录和详情。
6. 使用标志位置查询功能。
7. 展示统计分析和日报。
8. 展示标志字典管理和管理员维护能力。
