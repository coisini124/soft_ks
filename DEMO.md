# 课堂演示指南

本项目已经支持无真实模型的课堂演示模式。没有 `best.pt` 时，使用 mock 检测器也能跑通图片识别、视频识别、结构化解析、违规写入、历史记录、统计分析和日报导出。

## 1. 生成演示数据

在项目根目录运行：

```bash
python -B scripts/prepare_demo_data.py
```

脚本会自动完成：

- 使用管理员账号登录测试接口；
- 生成一条图片超速检测记录；
- 生成一条图片违停检测记录；
- 生成一条视频检测记录；
- 写入对应违规预警台账；
- 重建当天日报；
- 输出可展示的检测记录编号和统计数量。

## 2. 登录系统

默认管理员账号：

```text
admin / admin123
```

## 3. 推荐演示顺序

1. 工作台：展示图片上传、视频上传、摄像头抓拍、违规提醒。
2. 历史记录：查看检测记录、识别结果图、结构化标志详情。
3. 违规预警台账：展示超速、违停等违规类型、风险等级和处理状态。
4. 标志位置查询：按路段、方向、类型或经纬度查询标志信息。
5. 标志字典管理：管理员新增、编辑、删除、导入、导出交通标志。
6. 数据分析：展示标志分布、检测趋势、违规趋势、日报生成和导出。
7. 操作审计日志：展示管理员关键操作记录。

## 4. 真实模型说明

mock 模式只用于课堂流程演示，不代表真实识别准确率。若需要展示真实模型效果，将训练好的权重放到 `models/best.pt`，并把环境变量切换为：

```bash
DETECTOR_MODE=auto
MODEL_PATH=models/best.pt
```

真实准确率验收使用：

```bash
python -B scripts/evaluate_model.py --model best.pt --data path/to/data.yaml --split test --output reports/model_eval.json
```
